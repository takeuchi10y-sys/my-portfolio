# ポートフォリオサイト 設計書

本ドキュメントは、Next.js（フロント）・Express（バックエンド）・Prisma ORM・PostgreSQL・Firebase Authentication を用いて構築したポートフォリオサイトの設計情報をまとめたものです。  
画面構成、データモデル、API 仕様、セキュリティ方針、実装済み機能を記載しています。

---

## 1. 画面構成とアクセス権限

| 画面名               | URL        | アクセス権限           | 備考                                     |
| -------------------- | ---------- | ---------------------- | ---------------------------------------- |
| トップページ         | `/`        | 一般公開               | プロフィール、実績一覧、スキル一覧を表示 |
| コンタクトページ     | `/contact` | 一般公開               | 問い合わせフォーム                       |
| 管理者ログインページ | `/login`   | 一般公開               | Firebase Auth による認証                 |
| 管理画面トップ       | `/admin`   | 管理者のみ（認証必須） | 未認証時はガード処理でアクセス不可       |

---

## 2. データベース設計（Prisma Schema）

### Profile モデル

- `id`: Int（PK）
- `name`: String
- `title`: String
- `past`: String
- `present`: String
- `future`: String
- `avatarUrl`: String（デフォルト `/profile.jpg`）
- `skills`: String[]
- `projects`: String[]
- `createdAt`: DateTime
- `updatedAt`: DateTime

---

## 3. API エンドポイント設計

| メソッド | エンドポイント  | 認証 | 役割                                                              |
| -------- | --------------- | ---- | ----------------------------------------------------------------- |
| GET      | `/api/profile`  | 不要 | PostgreSQL からプロフィール情報を取得（404/500 ハンドリング済み） |
| GET      | `/api/projects` | 不要 | 実績一覧取得（モック → 拡張予定）                                 |
| POST     | `/api/contact`  | 不要 | 問い合わせ内容の受信（モック）                                    |

---

## 4. セキュリティ・環境変数管理

- Firebase Config と DB 接続情報（`DATABASE_URL`）は GitHub にコミットしない
- `.env` / `.env.local` は `.gitignore` により除外
- 過去に誤って追跡された `backend/.env` は `git rm --cached` により履歴から削除済み

---

## 5. 実装済み機能

### 5-1. フロント：Firebase SDK 導入（Issue #5）

- Firebase SDK を導入し、`frontend/lib/firebase.ts` を作成
- `.env.local` に認証情報を安全に保持

### 5-2. フロント：Firebase Auth ログイン（Issue #8）

- `signInWithEmailAndPassword` による管理者ログイン
- 認証後 `/admin` へリダイレクト

### 5-3. バックエンド：API 実装（Issue #12）

- Express + CORS 設定
- PrismaClient による PostgreSQL 連携
- `/api/profile` を本番データ対応にアップグレード
